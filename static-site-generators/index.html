Static site generators (SSGs) compiles simple text files (like markdown) into websites.

See also awesome static site generators.

Sort by programming languages

R

  • Bookdown : Write HTML, PDF, ePub, and Kindle books with R Markdown.

Ruby

  • Jekyll : The default SSG for GitHub pages.

Rust

  • mdBook : Create book from markdown files. Like Gitbook but implemented in Rust.
  • Zola : A fast static site generator in a single binary with everything built-in.

Javascript / npm

  • Docsify : Rendering Markdown files on-the-fly. Awesome docsify.
  • Gatsby.js : A blazing fast React-based SSG. Some javascript knowledge is required.
  • Hexo : A fast, simple & powerful blog framework written in NodeJS. Themes and plugins.

Go

  • Hugo : The fastest framework for building websites, written in Go. Themes.

Python

  • Jupyterbook : Building beautiful, publication-quality books and documents from jupytr notebooks.
  • MkDocs : MkDocs is a fast, simple and downright gorgeous static site generator that’s geared towards building project documentation. Material theme
  • Nikola : Static Site Generator written in Python.

Julia

  • Franklin.jl : Static site generation with live Julia code evaulation. Examples.
  • PkgPage.jl : Creating (package) front-pages, powered by Franklin.jl.
  • StaticWebPages.jl : Create academics and personal CV web-pages.

SSGs I used so far

Hugo

Hexo

Pros üëç

  • A huge set of plugins thanks to the npm ecosystem.
  • Plentiful Chinese resources.
  • Relatively fast. It builds a large website under 1 minute.

Cons üëé

  • Hundreds of nodeJS dependencies. If you are familiar with npm package management it might not be a problem.
  • For LaTeX math typesetting, you need to change the renderer to either the pandoc one or the markdown-it one.
  • Less intuitive asset management.

Themes

  • Next
  • fluid
  • butterfly

Docsify

Docsify renders Markdown files to HTML on-the-fly.

Pros üëç

  • Minimal dependency: just one index.html and some markdown files.
  • It skips the building step and renders Markdown file on-the-fly.
  • Handy extensions on awesome docsify.

Cons üëé

  • No support for tags, categories, and footnotes extensions.
  • The KaTeX extension does not support mhchem extension.

Others

  • Jekyll, the default GitHub pages SSG written in Ruby. Generating a large site is slower compared to either Hugo or Hexo, taking several minutes.
  • Nikola, a SSG written in Python with first-class support of Markdown (*.md), reStructuredText (*.rst) and Jupyter Notebook (*.ipynb) files.
  • JupyterBook, powered by the Sphinx python documentation generator, builds beautiful, publication-quality books and documents from Markdown (*.md), reStructuredText (*.rst) and Jupyter Notebook (*.ipynb) files.
  • Gatsby.js is a blazing fast React-based open-source framework for creating websites and apps. Some Javascript Node.js knowledge is required to build a site.

  1. In Hugo, you are able to override default layouts and settings by placing counterpart file(s) in your site folder. Hugo will look at your custom setting first without messing with the theme folder, which is much more friendly for Git submodules and theme updates. ↩︎
